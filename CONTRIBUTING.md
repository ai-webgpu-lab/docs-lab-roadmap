# Contributing

`ai-webgpu-lab`은 실험 저장소와 벤치마크 저장소를 분리하고, 모든 결과를 재현 가능한 형태로 남기는 것을 기본 원칙으로 합니다.

## 기본 원칙
- 저장소 하나는 하나의 질문 또는 하나의 책임을 갖습니다.
- 문서/실험/벤치/앱은 `docs/01-org-repo-map.md`의 네이밍 규칙을 따릅니다.
- 모든 실험 결과는 `RESULTS.md`와 raw JSON 결과를 함께 남깁니다.
- 브라우저, OS, 디바이스, 백엔드, 캐시 상태를 PR 또는 결과 문서에 명시합니다.

## 이 저장소에서 다루는 내용
- 조직 운영 문서
- 저장소 인벤토리와 우선순위
- GitHub Projects/issue/PR 운영 규칙
- 템플릿, 스키마, 스캐폴드

개별 실험 구현은 각 `exp-*`, `bench-*`, `app-*`, `shared-*` 저장소에서 진행합니다.

## 작업 흐름
1. 관련 이슈를 먼저 정리하거나 새 이슈를 생성합니다.
2. 변경 이유와 범위를 문서에 명시합니다.
3. 필요한 경우 `README`, 템플릿, 스키마, 프로젝트 문서를 함께 갱신합니다.
4. PR 템플릿 기준으로 실행 환경과 결과 근거를 남깁니다.

## PR 기준
- 요약은 무엇이 바뀌었는지와 왜 필요한지를 먼저 설명합니다.
- 운영 규칙에 영향이 있으면 영향 받는 저장소 또는 문서를 적습니다.
- 템플릿을 바꾸면 downstream 저장소 반영 계획을 함께 적습니다.
- 조직 정책 변경은 `docs-*` 문서 또는 ADR 문서와 연결합니다.
